# AGENTS.md

## Scope
- This repo is a Nix flake for machine and home-manager configuration, not a general app codebase.
- Start with `flake.nix` and `lib/mkSystem.nix`; that is the real wiring.

## Real Entry Points
- `flake.nix` exports only `nixosConfigurations.{laptop,gamer,work}` and `packages.x86_64-linux.{faugus-launcher,wagoapp}`.
- `lib/mkSystem.nix` constructs both normal NixOS hosts and the WSL host. It wires Home Manager into every host for user `stianrs`.
- Host-specific system config lives in `hosts/<hostname>/default.nix`.
- Host-specific home-manager config lives in `homes/stianrs/hosts/<hostname>.nix`.
- Shared host modules are split between `hosts/_modules/common`, `hosts/_modules/nixos`, and `hosts/_modules/wsl`.

## Package Boundaries
- `overlays/default.nix` injects custom packages into `pkgs` and adds `pkgs.unstable`.
- Do not assume every file in `pkgs/` is exported. The flake currently exposes only `faugus-launcher` and `wagoapp` because `pkgs/default.nix` only returns those attrs.

## Verification
- Inspect exported targets with `nix flake show --all-systems`.
- When changing a host, validate the exact host with `nix build .#nixosConfigurations.<host>.config.system.build.toplevel`.
- When changing a custom package, validate the exact package with `nix build .#faugus-launcher` or `nix build .#wagoapp`.
- This flake does not define a `checks` output; prefer targeted `nix build` commands over assuming `nix flake check` covers your change.
- PR CI currently runs `nix run nixpkgs#statix -- fix`; if you touch Nix files, run that and review the diff.

## Rebuild Workflow
- On machines that already use this repo, `programs.nh.flake` is pinned to `/home/stianrs/nix` in `hosts/_modules/nixos/system/nh/default.nix`.
- If you are rebuilding locally on one of those machines, prefer `nh os switch` instead of inventing a different wrapper.

## Generated And Secrets
- `pkgs/_sources/generated.nix` is generated by nvfetcher; do not edit it manually.
- Update nvfetcher inputs in `pkgs/nvfetcher.toml`, then from `pkgs/` run `nix-shell -p nvfetcher --command 'nvfetcher -l /tmp/nvfetcher-changes --keep-old'`.
- The repo uses `sops-nix`. Common config sets `sops.age.sshKeyPaths = [ "/etc/ssh/ssh_host_ed25519_key" ]`.
- `hosts/README.md` documents the workstation age-key bootstrap: `op document get "Age Key" --out-file ~/.config/age/keys.txt`.

## Current Host Facts
- Supported system in `flake.nix`: `x86_64-linux` only.
- `work` is the WSL host; it is built with `mkWslSystem`, not `mkNixosSystem`.
