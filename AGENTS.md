# AGENTS.md

## Scope
- This repo is a Nix flake for machine and Home Manager configuration, not an application codebase.
- Start with `flake.nix` and `lib/mkSystem.nix`; they define the real wiring.
- The flake only declares `x86_64-linux`.

## Entry Points
- `flake.nix` exports only `nixosConfigurations.{laptop,gamer,work}` and `packages.x86_64-linux.{faugus-launcher,wagoapp}`.
- `lib/mkSystem.nix` wires Home Manager into every host for user `stianrs`.
- System host entrypoints are `hosts/<hostname>/default.nix`.
- Home Manager entrypoint is `homes/stianrs/default.nix`; it imports `homes/stianrs/hosts/${hostname}.nix`.
- `work` is the WSL host. It is built with `mkWslSystem`, pulls in `hosts/_modules/wsl`, and passes `isWSL = true` into Home Manager.

## Module Layout
- Host files are intentionally thin. Most reusable behavior lives under `hosts/_modules/{common,nixos,wsl}` and `homes/_modules/**`.
- The repo uses a custom `modules.*` option tree in both NixOS and Home Manager. Add reusable toggles in `_modules` and enable them from host files instead of hardcoding more host-local config.
- Home Manager modules already receive `hostname`, `system`, `inputs`, and `isWSL`; reuse those args before adding new plumbing.

## Packages
- `overlays/default.nix` injects custom packages into `pkgs` and adds `pkgs.unstable`.
- Adding a file under `pkgs/` does not export it automatically. Only attrs returned from `pkgs/default.nix` become flake packages.
- `pkgs/_sources/generated.nix` is generated by nvfetcher; edit `pkgs/nvfetcher.toml` instead.

## Verification
- Inspect current outputs with `nix flake show --all-systems`.
- For host changes, build the exact target: `nix build .#nixosConfigurations.<host>.config.system.build.toplevel`.
- For exported packages, build the exact package: `nix build .#faugus-launcher` or `nix build .#wagoapp`.
- This flake does not define a meaningful `checks` output; use targeted builds instead of assuming `nix flake check` covers your change.
- If you touch Nix files, run `nix run nixpkgs#statix -- fix` and review the diff. CI runs that exact command.

## Local Rebuild
- On machines that already use this repo, `programs.nh.flake` is pinned to `/home/stianrs/nix` in `hosts/_modules/nixos/system/nh/default.nix`.
- For local rebuilds on those machines, prefer `nh os switch`.

## Generated And Secrets
- Secrets are handled with `opnix`.
- To refresh nvfetcher-managed sources, run from `pkgs/`: `nix-shell -p nvfetcher --command 'nvfetcher -l /tmp/nvfetcher-changes --keep-old'`.
