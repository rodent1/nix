# AGENTS.md

Nix flake managing NixOS hosts, a WSL host, and Home Manager configurations through flake-parts and import-tree.

## Architecture
- Pattern: dendritic flake-parts via `mightyiam/import-tree`. `flake.nix` is `mkFlake` plus `(inputs.import-tree ./modules)`, so every non-ignored `.nix` file under `modules/` is a flake-parts module loaded recursively.
- Formatter: `treefmt-nix`, exposed through `nix fmt` and the flake check.
- Systems: only `x86_64-linux` is declared.
- Key inputs: `nixpkgs`, `nixpkgs-unstable`, `flake-parts`, `import-tree`, `home-manager`, `nixos-wsl`, `nix-index-database`, `catppuccin`, `opnix`.

## Entry Points
- Root flake: `flake.nix`
- Host wiring helper: `modules/hosts/_lib.nix`
- NixOS host definitions: `modules/hosts/{gamer,laptop,work}.nix`
- NixOS host-local config: `hosts/<hostname>/default.nix`
- Home Manager shared wiring: `modules/home/_lib.nix`
- Home Manager user entrypoint: `modules/home/users/stianrs.nix`
- Standalone Home Manager outputs: `modules/home/configurations/stianrs.nix`

## Current Outputs
- `nixosConfigurations.{gamer,laptop,work}`
- `homeConfigurations.{gamer,laptop,work}`
- `packages.x86_64-linux.wagoapp`
- `overlays.{additions,nixpkgs-overlays,unstable-packages}`
- `homeModules.*` and `nixosModules.*` registries merged through flake-parts

## Directory Layout
- `modules/flake/*.nix`: flake-parts plumbing, systems, lib wiring, treefmt, Home Manager flake module import
- `modules/hosts/*.nix`: host registrations into `flake.nixosConfigurations.*`
- `modules/home/*.nix`: top-level Home Manager feature modules registered under `flake.homeModules.*`
- `modules/home/<feature>/**`: feature-local payload and child modules; ignored subtrees are prefixed with `_` when they should not be auto-loaded by import-tree
- `modules/home/users/stianrs.nix`: shared user profile and host-specific HM toggles for `gamer`, `laptop`, `work`
- `modules/nixos/features/*.nix`: explicitly registered NixOS feature modules
- `modules/nixos/legacy/*.nix`: temporary wrappers around still-unmigrated NixOS module trees under `hosts/_modules`
- `modules/overlays/*.nix`: one overlay per file via `flake.overlays.*`
- `modules/packages/default.nix`: package exports for `perSystem.packages`
- `hosts/_modules/{common,nixos,wsl}`: remaining reusable NixOS modules, still partly on the pre-dendritic layout
- `hosts/_modules/nixos/users/ssh.pub`: SSH key payload now used by the NixOS `stianrs` user module
- `pkgs/*.nix`: raw derivations and nvfetcher sources

## Module Pattern
- Flake-parts files export attrs like `flake.homeModules.<name>`, `flake.nixosModules.<name>`, `flake.overlays.<name>`, or `perSystem.packages.<name>`.
- Home Manager and NixOS feature modules still use the repo-local `modules.*` option tree internally.
- Imports that are part of runtime module composition belong inside the nested Home Manager or NixOS module value, not at the outer flake-parts file level.
- If a subtree is payload-only or should only be imported manually, put it under a path segment starting with `_` so import-tree ignores it.

## Host Composition
- `modules/hosts/_lib.nix` builds both regular NixOS hosts and the WSL host.
- `work` is the WSL host. It includes `inputs.nixos-wsl.nixosModules.default`, `hosts/_modules/wsl`, and passes `isWSL = true` into Home Manager.
- Home Manager integration on NixOS uses `inputs.home-manager.nixosModules.home-manager` with:
	- `useUserPackages = true`
	- `useGlobalPkgs = true`
	- shared HM modules from `modules/home/_lib.nix`
	- `users.stianrs = self.homeModules.stianrs`

## Packages And Overlays
- `modules/overlays/unstable-packages.nix` provides `pkgs.unstable`.
- `modules/overlays/additions.nix` exposes custom packages into `pkgs`.
- Exported packages are registered from `modules/packages/default.nix`.
- `pkgs/_sources/generated.nix` is generated by nvfetcher; edit `pkgs/nvfetcher.toml` instead of the generated files.

## Secrets
- Secrets are handled with `opnix` / `onepassword-secrets`.
- Never commit plaintext secrets.

## Verification
- Inspect current outputs with `nix flake show --all-systems`.
- For Home Manager changes, prefer `nix build .#homeConfigurations.<host>.activationPackage --no-link`.
- For host changes, prefer `nix build .#nixosConfigurations.<host>.config.system.build.toplevel --no-link`.
- For exported packages, prefer `nix build .#wagoapp`.
- When updating `pkgs/nvfetcher.toml` or generated sources, `nix flake check --no-build` is the lightweight validation path.

## Local Rebuild
- On machines that already use this repo, `programs.nh.flake` is pinned to `/home/stianrs/nix` in `hosts/_modules/nixos/system/nh/default.nix`.
- For local rebuilds on those machines, prefer `nh os switch`.

## Adding Things
1. New Home Manager feature: add a flake-parts file under `modules/home/` exporting `flake.homeModules.<name> = { ... }: { ... };`.
2. New NixOS feature: add a flake-parts file under `modules/nixos/` exporting `flake.nixosModules.<name>`.
3. New host: add a `modules/hosts/<host>.nix` registration file and a matching `hosts/<host>/default.nix` host-local module.
4. New overlay: add `modules/overlays/<name>.nix` exporting `flake.overlays.<name>`.
5. New package: register it from `modules/packages/default.nix` or another `modules/packages/*.nix` file.
