# AGENTS.md

## Scope
- This repo is a Nix flake for machine and Home Manager configuration, not an application codebase.
- Start with `flake.nix` and `modules/flake/`; they define the real wiring.
- The flake only declares `x86_64-linux`.

## Entry Points
- `flake.nix` contains inputs and the minimal flake-parts/import-tree bootstrap.
- The public outputs are `nixosConfigurations.{laptop,gamer,work}`, `packages.x86_64-linux.{flate,kubectl-kopiur,wagoapp}`, and the treefmt check/formatter.
- `modules/flake/hosts.nix` wires embedded Home Manager into every host for user `stianrs`.
- Host metadata and host-specific NixOS modules live in `modules/hosts/<hostname>.nix`.
- `work` is the WSL host and merges `rodent.nixosModules.wsl`.

## Module Layout
- Every maintained `.nix` file under `modules/` is an automatically imported flake-parts module.
- Underscore-prefixed path segments are ignored by `import-tree` and hold generated hardware, package recipes/sources, assets, or application configuration.
- Reusable NixOS behavior merges into `rodent.nixosModules.default`; WSL-only and host-only behavior merge into `.wsl` or `.<hostname>`.
- Reusable Home Manager behavior merges into `rodent.homeModules.default`; host-only behavior merges into `.<hostname>`.
- The existing lower-level NixOS and Home Manager feature options remain under `modules.*`.

## Packages
- `modules/flake/nixpkgs.nix` composes package additions, stable aliases, and `pkgs.unstable` in that order.
- Add a package recipe under `modules/packages/_recipes/`, register it with `rodent.packageRecipes`, and export the overlay package from a module under `modules/packages/`.
- `modules/packages/_sources/generated.nix` is generated by nvfetcher; edit `modules/packages/nvfetcher.toml` instead.

## Verification
- Inspect current outputs with `nix flake show --all-systems`.
- For host changes, prefer building the exact target locally: `nix build .#nixosConfigurations.<host>.config.system.build.toplevel`.
- For exported packages, prefer building the exact package locally: `nix build .#flate`, `nix build .#kubectl-kopiur`, or `nix build .#wagoapp`.
- CI uses `nix flake check` as a general flake validation step and also builds all exported `nixosConfigurations` and `packages.x86_64-linux` outputs through the cache-building workflow.
- When updating `modules/packages/nvfetcher.toml` or generated sources, `nix flake check --no-build` matches the lightweight validation used by the nvfetcher automation.

## Local Rebuild
- On machines that already use this repo, `programs.nh.flake` is pinned to `/home/stianrs/nix` in `modules/nixos/system/nh.nix`.
- For local rebuilds on those machines, prefer `nh os switch`.

## Generated And Secrets
- Secrets are handled with `opnix`.
- To refresh nvfetcher-managed sources, run from `modules/packages/`: `nix-shell -p nvfetcher --command 'nvfetcher -l /tmp/nvfetcher-changes --keep-old'`.
